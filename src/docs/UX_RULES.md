# BigFana - UX Rules

# UX Philosophy

BigFana is a MOBILE-FIRST WEB.APP.

This is NOT a desktop-first dashboard.

Desktop experiences should feel like expanded versions of the mobile product.

The platform should behave like:

- a premium consumer app
- a sports streaming product
- a modern fintech app
- an immersive fan platform

NOT like:

- an ERP
- an admin panel
- a traditional enterprise dashboard
- a spreadsheet-based interface

---

# Core UX Principles

The experience should prioritize:

- clarity
- emotional immersion
- touch ergonomics
- simplicity
- hierarchy
- storytelling
- responsiveness
- movement
- focus

---

# Mobile-First Rules

All interactions must be designed mobile-first.

Always prioritize:

- touch targets
- thumb reach
- gesture-friendly flows
- overlays
- fullscreen interactions
- immersive mobile patterns

Desktop should preserve the same emotional flow.

---

# Touch Ergonomics

## Minimum Touch Areas

Interactive elements should be large and comfortable.

Preferred:

- 44px minimum
- 48px+ preferred
- generous spacing between actions

Avoid:

- tiny icons
- dense action groups
- small clickable labels

---

# Navigation Philosophy

Navigation should feel:

- immersive
- soft
- direct
- predictable
- emotional

Avoid:

- excessive menu nesting
- enterprise sidebars
- admin complexity
- crowded top navigation

---

# Mobile Navigation

Preferred patterns:

- bottom navigation
- sticky navigation
- gesture-friendly layouts
- floating actions
- immersive fullscreen transitions

Avoid:

- tiny desktop nav patterns adapted poorly to mobile

---

# Sidebar Rules

Desktop sidebar should feel:

- lightweight
- atmospheric
- modern
- minimal

Avoid:

- heavy admin sidebars
- aggressive separators
- complex enterprise tree menus

---

# Progressive Disclosure

Do not overload screens with information.

Preferred:

- layered flows
- expandable sections
- sheets
- progressive reveal
- focused actions

Avoid:

- giant forms
- crowded dashboards
- too many simultaneous metrics

---

# One Primary Action Rule

Every screen should have:

- one dominant action
- one clear focus

Avoid competing CTAs.

---

# Bottom Sheet Philosophy

Bottom sheets are core UX patterns in BigFana.

Preferred uses:

- filters
- date selection
- ticket purchase
- reward redemption
- quick actions
- fan interactions
- live stats
- onboarding steps
- settings
- sponsor actions

Sheets should feel:

- immersive
- touch-native
- premium
- cinematic
- layered

Inspired by:

- Airbnb
- Apple Maps
- Uber
- Spotify
- modern iOS apps

---

# Modal Philosophy

Desktop modals should feel:

- spacious
- premium
- calm
- cinematic

Mobile should often prefer:

- fullscreen flows
- bottom sheets
- immersive overlays

Avoid tiny desktop popups on mobile devices.

---

# Date Picker Rules

Date selection should be mobile-native.

Preferred:

- bottom sheets
- wheel pickers
- calendar cards
- segmented quick selections
- fullscreen mobile flows

Avoid:

- tiny desktop calendar popovers
- small date dropdowns
- enterprise calendar widgets

---

- horizontal scrolling
- snap interactions

- immersive content rails

- live matches
- rewards
- featured sponsors
- stories
- players

- tickets
- highlights

---

# Carousel Rules

Preferred:

- smooth snap scrolling

- Buy Ticket

- Participate
- Vote

- sticky bottom CTA
- floating actions
- persistent mobile actions

---

# Feed Philosophy

- alive
- emotional
- immersive
- social
- breathable

Avoid:

- dense social feeds

- repetitive cards

# Form Philosophy

- clean
- spacious

# Form Rules

- large inputs
- clear hierarchy
- progressive forms

The product should heavily use:

- switches
- segmented controls
- chips
- pills
- toggles

- small desktop checkboxes

---

# Search Experience

Search should feel immersive.

Preferred patterns:

- fullscreen search
- command palette style overlays
- floating search experiences
- predictive suggestions

Avoid:

- tiny search bars
- enterprise filter panels

---

# Empty States

Empty states should feel:

- premium
- calm
- guided
- emotional

They should help users understand:

- what to do next
- why the area matters
- how to engage

Avoid generic placeholder messages.

---

# Loading States

Loading should feel:

- soft
- premium
- ambient
- non-blocking

Preferred:

- skeleton loaders
- shimmer
- progressive reveal
- smooth transitions

Avoid:

- spinning loaders everywhere
- abrupt layout jumps

---

# Motion Philosophy

Motion should feel:

- cinematic
- physical
- premium
- restrained

Avoid:

- gaming animations
- bouncing UI
- flashy motion
- excessive scaling

Preferred:

- opacity transitions
- soft translate motion
- ambient glow transitions
- subtle hover elevation

---

# Feedback System

Feedback should feel:

- immediate
- clear
- soft
- human

Preferred:

- toasts
- inline confirmation
- subtle haptics-inspired motion

Avoid:

- harsh alerts
- technical system messages

---

# Safe Area Rules

The platform must respect:

- mobile notches
- bottom safe areas
- gesture zones
- sticky bottom navigation spacing

Especially on iOS devices.

---

# Emotional UX Philosophy

BigFana is an emotional platform.

The interface should create feelings of:

- anticipation
- belonging
- excitement
- participation
- live energy
- identity

Without becoming:

- chaotic
- childish
- gambling-oriented
- gaming-noisy

---

# Density Rules

Avoid:

- crowded dashboards
- excessive widgets
- too many charts
- spreadsheet layouts

Prefer:

- focused modules
- storytelling
- breathing room
- emotional hierarchy
- immersive sections

---

# Final UX Standard

Every interaction should feel:

- premium
- touch-friendly
- cinematic
- modern
- emotional
- believable
- fast
- human

If something feels like an enterprise dashboard,
simplify it and make it feel more immersive and mobile-native.
