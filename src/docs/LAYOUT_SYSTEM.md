# BigFana - Layout System

# Layout Philosophy

BigFana is a mobile-first web.app.

Layouts should feel:

- immersive
- spacious
- cinematic
- emotional
- modular
- breathable

The platform should NOT feel like:

- an enterprise admin dashboard
- an ERP
- a spreadsheet application
- a dense analytics panel

Desktop should behave like an expanded version of the mobile experience.

---

# Core Layout Principles

Layouts should prioritize:

- breathing room
- hierarchy
- storytelling
- modular sections
- emotional focus
- touch ergonomics
- immersive navigation

Avoid:

- crowded grids
- tiny widgets
- dense panels
- excessive simultaneous information

---

# Application Shell

The application shell is one of the most important parts of BigFana.

It defines:

- navigation
- atmosphere
- spacing rhythm
- visual consistency
- immersive structure

---

# Desktop Layout

## Structure

Desktop layout consists of:

- collapsible sidebar
- top navigation/header
- centered content area
- immersive hero sections
- modular content blocks

---

## Desktop Philosophy

Desktop should NOT feel like a traditional admin panel.

Preferred feeling:

- immersive platform
- sports media experience
- premium operating system
- emotional fan ecosystem

Avoid:

- corporate dashboard structure
- multi-column enterprise layouts
- excessive widget density

---

## Content Width

Content should not stretch infinitely.

Preferred:

- centered layouts
- readable width
- breathing room on ultra-wide displays

Avoid:

- edge-to-edge spreadsheet layouts
- giant stretched dashboards

---

# Mobile Layout

## Philosophy

Mobile is the primary experience.

Layouts should feel:

- native-like
- immersive
- fluid
- thumb-friendly
- cinematic

- immersive hero areas

- bottom sheets

---

## Mobile Spacing

Mobile layouts should preserve:

- large touch areas

- breathing room
- strong hierarchy

- tiny stacked cards

- overloaded feeds

# Dashboard Density Rules

BigFana is NOT a dense analytics product.

Avoid:

- giant metrics walls
- excessive chart dashboards
- spreadsheet composition
- admin-heavy density

Prefer:

- storytelling
- emotional hierarchy
- focused modules
- premium sections

---

# Safe Area Rules

Layouts must respect:

- iOS safe areas
- bottom navigation spacing
- mobile gesture zones
- sticky action spacing

---

# Responsive Philosophy

Responsive design should NOT simply shrink desktop layouts.

Instead:

- mobile-first structure
- progressive enhancement
- immersive scaling
- layout adaptation

Desktop should preserve mobile emotional behavior.

---

# Motion-Aware Layout

Layouts should support:

- smooth transitions
- cinematic reveals
- soft hover motion
- layered movement
- ambient interaction

Avoid:

- abrupt layout jumps
- chaotic transitions
- excessive animation

---

# Multi-Tenant Layout Support

Layouts must support:

- dynamic club branding
- configurable hero imagery
- sponsor integrations
- dynamic campaign modules

Without breaking:

- spacing consistency
- hierarchy
- readability
- emotional atmosphere

---

# Final Layout Standard

Every layout should feel:

- premium
- immersive
- cinematic
- emotional
- spacious
- touch-native
- believable
- modern sports-tech

If a layout starts feeling like an enterprise dashboard,
simplify it and make it more immersive and editorial.
