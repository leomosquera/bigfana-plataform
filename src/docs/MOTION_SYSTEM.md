# BigFana - Motion System

# Motion Philosophy

Motion in BigFana should feel:

- cinematic
- premium
- emotional
- restrained
- physical
- immersive

Motion should support:

- hierarchy
- focus
- immersion
- anticipation
- continuity
- responsiveness

The platform should NEVER feel:

- flashy
- gaming-heavy
- chaotic
- cartoonish
- overanimated
- distracting

---

# Core Motion Principles

Motion should:

- guide attention
- reinforce hierarchy
- improve clarity
- create emotional atmosphere
- support immersion
- make interactions feel alive

Avoid motion that exists only for decoration.

---

# Motion Tone

BigFana motion should feel closer to:

- Apple
- Linear
- Airbnb
- modern streaming apps
- premium sports broadcasts

NOT:

- gaming HUDs
- esports overlays
- crypto dashboards
- flashy SaaS motion

---

# Motion Speed

Preferred motion timing:

- fast enough to feel responsive
- soft enough to feel premium

General range:

- 150ms → micro interactions
- 200ms → buttons & hover
- 250-350ms → sheets & overlays
- 400-600ms → hero transitions

Avoid:

- extremely slow motion
- hyper-fast twitchy motion

---

# Easing Philosophy

Preferred easing:

- soft
- natural
- physical
- cinematic

Avoid:

- exaggerated bounce
- elastic cartoon motion
- playful spring chaos

Preferred feel:

- smooth acceleration
- calm deceleration
- premium inertia

---

# Hover Motion

Hover states should feel:

- subtle
- premium
- restrained

Preferred:

- slight elevation
- soft glow
- minimal translateY
- opacity refinement
- soft scale

Avoid:

- dramatic scaling
- glowing explosions
- flashy hover effects

---

# Button Motion

Buttons should feel:

- responsive
- tactile
- modern

Preferred interactions:

- subtle scale down on tap
- soft hover lighting
- smooth background transition
- subtle glow changes

Avoid:

- bouncing
- aggressive pulse
- flashy gradients

---

# Card Motion

Preferred:

- slight elevation on hover
- soft shadow expansion
- subtle parallax layers
- ambient glow refinement

Avoid:

- aggressive floating
- dramatic transforms
- card flipping gimmicks

---

# Page Transitions

- cinematic
- soft
- immersive

---

# Scroll Philosophy

Scrolling is part of the emotional experience.

Preferred:

- smooth rhythm
- breathing room
- layered content reveal
- immersive flow

Avoid:

- cramped layouts
- abrupt scroll jumps
- overloaded feeds

---

# Motion Density Rules

Not every component should animate.

Motion should create hierarchy.

Important areas receive more motion attention:

- hero sections
- live modules
- primary CTA
- sheets
- overlays

Utility areas should remain calmer.

---

# Ambient Motion

BigFana may use subtle ambient motion:

- moving gradients
- lighting shifts
- glow breathing
- layered background movement

Always subtle.

Avoid visible gimmicks.

---

# Accessibility Rules

Motion should remain comfortable.

Support reduced motion preferences when possible.

Avoid:

- flashing
- aggressive movement
- excessive motion loops

---

# Technical Motion Rules

Preferred tools:

- Framer Motion
- CSS transitions
- GPU-friendly transforms

Prefer:

- opacity
- transform
- translate
- scale (subtle)

Avoid expensive layout animations when possible.

---

# Final Motion Standard

Motion should feel:

- premium
- cinematic
- immersive
- emotional
- restrained
- physical
- believable

If motion feels flashy, gaming-like, or distracting,
reduce it and simplify it.
